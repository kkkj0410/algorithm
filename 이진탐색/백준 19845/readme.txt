## 문제 유형

- 이진 탐색

## 문제 접근

- 답
    - 파괴 가능한 네모 개수
- 주어지는 정보
    - 레이저의 위치
    - 레이저가 네모를 파괴하는 방향
    
    ⇒ 해당 레이저를 기점으로 파괴 가능한 네모의 개수를 구할 수 있음
    
- 활용
    - x 좌표의 파괴 가능한 네모 개수는?
        - 현재 x 좌표의 전체 네모 개수 - 현재 x 좌표의 네모 개수
    - y 좌표의 파괴 가능한 네모 개수는?
        - 완전 탐색
            - 현재 y좌표를 기점으로, 위에도 네모가 있다면 count++
                - 시간 복잡도 = 25000*25000 ⇒ 실패
        - 이분 탐색
            - 이분 탐색으로 맨 위에 있는 네모의 개수를 찾음
                - 시간 복잡도 = 25000*log25000 = 1억 이하 ⇒ 성공 가능
