## 문제 유형

- 최단 거리 알고리즘

## 문제 접근

- 답
    - 최소 택시 요금
- 주어지는 정보
    - 목적지 개수 최대 200
- 후보
    - 완전탐색
        - 불가능
            - 각 목적지에 따른 모든 경우의 수를 구한다고 하자
            - 대략 200!로 경우의 수가 구성되는데, 한참 초과됨
    - dijkstra 알고리즘
        - 가능
        - 방법
            - v[] : start 지점에 대한 최단 거리 구성
            - curV[] : v 배열의 각 지점에 대해서 또 다시 최단 거리 구성
            - v[i] + curV[a] + curV[b] 지점의 최소값을 return
